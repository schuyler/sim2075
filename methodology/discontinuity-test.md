# Discontinuity Test

Five questions that require judgment. Answer before marking any event complete.

---

## 1. Is this actually a discontinuity?

For each resolution: could these state variable trajectories plausibly be reached through 10 years of gradual drift?

- If yes → not a discontinuity. Remove the resolution or raise the threshold.
- "More repressive," "more isolated," "more unstable" without structural change = sub-threshold stress.
- Test case: Tiananmen 1989. Extraordinary measures, but same regime governed same territory afterward. Not a discontinuity.

## 2. Does the probability match the structure?

Given your pressure estimate and threshold distance, does the probability make sense?

- If pressure is at 35 and threshold is at 85, why is annual probability above 1%?
- If you're modeling genuine regime change, why is it more likely than other regime changes?
- Check: would you bet at these odds?

## 3. Are the resolutions actually different?

Each resolution should produce qualitatively different state trajectories, not variations on a theme.

- "Mild collapse" vs. "severe collapse" → probably one resolution with uncertainty in magnitude
- "Transformation" vs. "collapse" → genuinely different outcomes, keep both
- If you can't clearly distinguish the aftermath branches, merge the resolutions

## 4. What's the strongest case against this specification?

State it explicitly. If you can't articulate what's wrong or uncertain, you haven't thought hard enough.

- Which assumption, if wrong, breaks the estimate?
- What would make you revise the probability by 2x or more?
- What does the skeptic say?

## 5. Would another analyst reach similar conclusions?

Given the same evidence and methodology, would someone else produce a similar specification?

- If the estimate depends on your intuition rather than documented reasoning, it's not auditable
- If two analysts would disagree by more than 2x, the uncertainty bounds are too narrow

---

## Usage

Run this before the format checklist. A specification that passes format checks but fails these questions is wrong in ways that matter more than missing fields.

If you can't answer "yes" to questions 1-3 and "explicitly" to questions 4-5, the specification isn't ready.
