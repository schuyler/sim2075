# ℹ️ FINDINGS VALID — FRAMING REVISED

**The empirical findings in this document remain valid. The interpretive framing has been revised.**

This analysis correctly identified five asymmetries in how discontinuities behave:
1. Entropy/default resolution (systems decay toward disorder)
2. Coordination requirements (collective action harder than unilateral action)
3. Technology direction (historical S-curves mostly beneficial — but survivorship bias caveat)
4. Reversibility (destruction more permanent than construction)
5. Visibility (gradual improvements not perceived as "events")

**What changed:**

The original framing posited "positive" and "negative" as event properties, leading to proposals for opportunity factors and valence-based asymmetric treatment. This was wrong.

**Revised understanding:**
- Events have **impact vectors**, not valences
- The asymmetries are about **causal structure** and **impact type**, not event valence
- Type 2 (threshold) events resolve negatively by default because entropy; positive resolution requires Type 3 (agency)
- Durability depends on **what kind of impact** (mortality, agreements, adoption), not whether it's "good" or "bad"

See [[methodology/integrated-event-state-framework]] for how these findings are incorporated into the model.

# Positive-Negative Discontinuity Asymmetry Analysis

## Overview

This note documents an empirical finding: positive and negative discontinuities have systematically different causal structures in the historical record. This asymmetry has implications for how we model each type and suggests our current symmetric treatment may introduce bias.

**Foundation**: This analysis builds on:
- [[methodology/discontinuity-causal-typology]] - The four-type causal framework
- [[research/positive-discontinuities-historical]] - Historical positive event inventory

## Distribution Across Causal Types

### Negative Discontinuities

| Type | Historical Examples | Projected Events in Catalog |
|------|---------------------|----------------------------|
| **Type 1 (Stochastic)** | Pandemic emergence, major earthquakes, some financial crises | Severe pandemic, oil supply shock |
| **Type 2 (Threshold)** | Hyperinflations, state failures (Syria, Venezuela, Somalia), currency crises, environmental collapse | AMOC collapse, Amazon tipping, Pakistan failure, Egypt failure, most climate events, most state failures |
| **Type 3 (Contingent)** | Iraq War, specific coups, deliberate escalations | Taiwan conflict, US-China war, some regime changes |
| **Type 4 (S-curve)** | *Rare historically* | AI disruption (negative scenario), cyber cascade |

**Pattern**: Negative events cluster heavily in **Type 2**. Systems accumulate pressure and cross thresholds into failure states.

### Positive Discontinuities

| Type | Historical Examples | Potential Catalog Events |
|------|---------------------|-------------------------|
| **Type 1 (Stochastic)** | Scientific breakthroughs, serendipitous discoveries | Fusion breakthrough, cancer cure, breakthrough discovery |
| **Type 2 (Threshold)** | End of Cold War, Velvet Revolutions, Latin American democratization (pressure → positive resolution) | *Underrepresented* |
| **Type 3 (Contingent)** | Good Friday Agreement, South Africa transition, post-2008 coordination, German reunification, Camp David Accords | US-China accommodation, Taiwan peaceful resolution, major climate agreement, Middle East breakthrough |
| **Type 4 (S-curve)** | Internet adoption, mobile revolution, solar/wind cost collapse, Green Revolution, mRNA platform, shale revolution | Accelerated decarbonization, AI productivity gains, energy storage breakthrough, EV adoption |

**Pattern**: Positive events cluster in **Types 3 and 4**. Coordination successes and technology adoption curves dominate.

### Summary Distribution

| Type | Negative Events | Positive Events |
|------|-----------------|-----------------|
| Type 1 (Stochastic) | Common | Uncommon |
| Type 2 (Threshold) | **Dominant** | Present but often requires Type 3 to resolve positively |
| Type 3 (Contingent) | Present | **Disproportionately represented** |
| Type 4 (S-curve) | Rare (mostly potential) | **Dominant for technology** |

## Five Identified Asymmetries

### Asymmetry 1: Entropy/Default Resolution

**Finding**: Negative Type 2 events can occur "by themselves"—systems decay, pressure accumulates, thresholds are crossed. The Second Law of Thermodynamics works against complex systems.

Positive Type 2 outcomes typically require *agency intervention* to convert accumulated pressure into good outcomes rather than bad.

**Evidence**:
| Accumulated Pressure Situation | Resolution | Type 3 Intervention Required? |
|-------------------------------|------------|------------------------------|
| Cold War tension | Peaceful end | Yes (Gorbachev decisions) |
| Yugoslav dissolution | Violent fragmentation | No (default outcome) |
| Syrian civil war | State collapse | No (default outcome) |
| South African apartheid | Peaceful transition | Yes (Mandela/de Klerk) |
| Czechoslovak communism | Velvet Revolution | Yes (coordinated resistance + regime non-violence) |
| Venezuelan economic pressure | Collapse | No (default outcome) |

**Assessment**: When Type 2 pressure accumulates, negative resolution appears to be the default (~60-70% of cases). Positive resolution requires active Type 3 coordination.

**Implication**: Treating positive and negative Type 2 events symmetrically overweights positive outcomes. The probability of positive resolution should be conditional on coordination/agency factors.

---

### Asymmetry 2: Coordination Requirements

**Finding**: Negative Type 3 events typically require *one* actor to defect, err, or escalate. Positive Type 3 events typically require *multiple* actors to coordinate successfully.

**Evidence**:
| Event Type | Actors Required | Failure Mode |
|------------|-----------------|--------------|
| Starting a war | 1 (attacker) | Single decision |
| Peace agreement | 2+ (all parties) | Any party defection |
| Trade war | 1 (initiator) | Single decision |
| Trade deal | 2+ (all parties) | Any party rejection |
| Coup | Small group | Coordination of plotters |
| Democratic transition | Regime + opposition + military | Multiple veto players |

**Game-Theoretic Framing**:
- Negative outcomes: Often achievable through unilateral action or coordination failure
- Positive outcomes: Often require solving coordination games, prisoner's dilemmas, or collective action problems

**Assessment**: The coordination threshold for positive outcomes is systematically higher than for negative outcomes.

**Implication**: Positive Type 3 events may need a "coordination success" factor we don't currently model. Current factors capture shared *stress*; we lack factors capturing shared *opportunity* or *alignment* that would enable coordination.

---

### Asymmetry 3: Technology Direction

**Finding**: Technology S-curves (Type 4) have been overwhelmingly positive historically. Major negative technology discontinuities remain potential rather than realized.

**Evidence**:
| Technology S-Curve | Outcome | Status |
|-------------------|---------|--------|
| Internet/WWW | Positive (information access, productivity) | Realized |
| Mobile phones | Positive (connectivity, financial inclusion) | Realized |
| Solar/wind | Positive (cost decline, deployment) | Realized, ongoing |
| mRNA vaccines | Positive (rapid development platform) | Realized |
| Green Revolution | Positive (yields, food security) | Realized |
| Nuclear weapons | Mixed (deterrence vs. risk) | Ongoing |
| AI | Uncertain | In progress |
| Social media | Mixed (connection vs. harms) | Realized, ongoing |
| Synthetic biology | Uncertain | Early stage |

**Potential Negative S-Curves (Not Yet Realized)**:
- AI catastrophe scenarios
- Engineered pandemic
- Autonomous weapons proliferation
- Surveillance state technology

**Assessment**: Realized technology S-curves show ~80%+ positive or mixed outcomes. Purely negative technology S-curves are speculative.

**Possible Explanations**:
1. **Survivorship bias**: We're here because negative tech discontinuities (nuclear war, engineered pandemic) haven't occurred
2. **Selection effect**: Technologies that succeed and diffuse tend to be ones that provide value
3. **Genuine asymmetry**: Technological progress tends to solve more problems than it creates

**Implication**: Our catalog may overweight potential negative technology events relative to historical base rates. Technology events might be better modeled as trend acceleration (baseline adjustment) rather than discrete shocks.

---

### Asymmetry 4: Reversibility

**Finding**: Negative discontinuities tend toward irreversibility. Positive discontinuities (except technology adoption) are often fragile and can be lost.

**Evidence**:

| Discontinuity Type | Examples | Reversibility |
|-------------------|----------|---------------|
| **Negative: Deaths** | Pandemic mortality, conflict deaths | Irreversible |
| **Negative: Destruction** | Infrastructure damage, institutional collapse | Slow/costly to reverse |
| **Negative: State failure** | Somalia, Syria, Libya | Years to decades to reconstitute |
| **Negative: Environmental** | Species extinction, some tipping points | Often irreversible |
| **Positive: Peace agreement** | Good Friday (holding), Oslo (collapsed) | Fragile, ~20-30% reversal |
| **Positive: Democratization** | Arab Spring (mostly reversed), Eastern Europe (mixed) | Significant reversal rates |
| **Positive: Development** | Venezuela (reversed), Argentina (cycles) | Can be lost |
| **Positive: Health gains** | Antibiotic resistance eroding gains | Reversible |
| **Positive: Tech adoption** | Internet, mobile | Rarely reverses once saturated |

**Quantitative Assessment**:
| Positive Discontinuity Type | Durability (30-year horizon) |
|----------------------------|------------------------------|
| Peace agreements | ~70-80% durable |
| Democratic transitions | ~60-70% durable |
| Economic development | ~70-80% durable (but cycles) |
| Technology adoption | ~95%+ durable |

**Implication**: Impact duration should be modeled asymmetrically:
- Negative events: Permanent or very slow recovery
- Positive events (non-technology): Require maintenance; decay probability each period
- Positive events (technology): Permanent once established

---

### Asymmetry 5: Visibility and Salience

**Finding**: Negative events are salient, sudden, and extensively covered. Positive changes accumulate gradually and are under-perceived.

**Evidence**:
| Change | Magnitude | Media Coverage | Public Perception |
|--------|-----------|----------------|-------------------|
| Global extreme poverty decline (1990-2020) | 1 billion lifted out | Minimal | Most people unaware |
| Child mortality decline | 7M fewer deaths/year | Occasional | Underestimated |
| Solar cost decline | 90%+ over 15 years | Moderate | Underestimated pace |
| Any major terrorist attack | Dozens to thousands dead | Extensive | Overestimated threat |
| Any major financial crisis | Significant but recoverable | Extensive | Salient for decades |

**Psychological Basis**:
- Negativity bias: Negative events more salient and memorable
- Availability heuristic: Dramatic events overweighted in probability judgments
- News selection: "If it bleeds, it leads"

**Causal Basis**:
- Failures *can* be sudden (threshold crossing)
- Successes *often* require sustained effort (S-curves, maintenance)
- Sudden positive events (breakthroughs) are less common than sudden negative events (disasters)

**Implication**: Expert forecasts and event catalogs may systematically underweight positive discontinuities because gradual improvements don't register as "events." A 90% cost decline over 15 years is genuinely discontinuous relative to baseline expectations, but doesn't feel like a discrete event.

## Quantitative Summary

| Asymmetry | Direction | Magnitude | Confidence |
|-----------|-----------|-----------|------------|
| Entropy/Default | Negative resolution favored for Type 2 | ~60-70% negative default | Medium |
| Coordination | Higher bar for positive outcomes | ~2-3x more actors needed | Medium-High |
| Technology | Positive S-curves dominate historically | ~80%+ positive realized | Medium (survivorship?) |
| Reversibility | Negative more durable | ~2x longer duration | Medium-High |
| Visibility | Positive underweighted | ~2-3x perception gap | High |

## Implications for Model Validity

### Current Model Assumptions

The factor-correlation model implicitly assumes:
1. Positive and negative events are symmetric in structure
2. Factor loadings work the same way for both
3. Events have similar duration distributions
4. Base rates can be estimated symmetrically

### Validity Assessment

| Assumption | Validity Given Asymmetries |
|------------|---------------------------|
| Symmetric structure | **Questionable** - Different types dominate |
| Symmetric factor loadings | **Partially valid** - Stress factors may not capture coordination opportunity |
| Similar duration | **Invalid** - Asymmetric reversibility |
| Symmetric estimation | **Questionable** - Technology especially may be underweighted |

### Expected Bias Direction

If asymmetries are real and unaddressed:
- **Negative tail**: Approximately correct (Type 2 dominates, factors capture stress)
- **Positive tail**: Likely underweighted (coordination not modeled, technology underestimated)
- **Net bias**: Pessimistic relative to true distribution

---

## Recommended Model Modifications

Based on this analysis, see [[methodology/asymmetric-modeling-proposal]] for specific recommendations:

1. Add coordination/opportunity factors for positive Type 3 events
2. Model technology as baseline trend acceleration rather than discrete events
3. Implement asymmetric duration (positive events require maintenance)
4. Review probability estimates for technology events against S-curve base rates

---

## Limitations and Uncertainties

1. **Small sample**: Historical positive discontinuities are limited; patterns may not generalize
2. **Survivorship bias**: We can only observe histories where catastrophic negatives didn't occur
3. **Hindsight bias**: Positive outcomes may look more contingent in retrospect than they were
4. **Selection of events**: The events we chose to analyze may not be representative
5. **Causal attribution**: Type classification involves judgment; others might classify differently
